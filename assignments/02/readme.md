# **Actividad 2: Resolución de 10 Problemas en Ensamblador ARM64**

**Descripción de la actividad:**

Deberás seleccionar y resolver **10 problemas de ensamblador** de la lista disponible en el repositorio de GitHub: [https://github.com/tectijuana/problemasresolverporcomputadora](https://github.com/tectijuana/problemasresolverporcomputadora). 

Los programas deben ser implementados en dos lenguajes, el promeero es el CSHARP Consola y el otro es ensamblador para arquitectura **ARM64 bits**. 

---

**Rúbrica de Evaluación (cada fase vale 25% del total):**

1. **Análisis y Diseño del Algoritmo (25%)**
   - Comprender cada problema y diseñar la lógica o algoritmo necesario para resolverlo.
   - Presentar un **pseudocódigo o diagrama de flujo** para cada problema el codigo Mermaid (Via chatgtp) solicitar ese formato

2. **Implementación en Ensamblador ARM64 (25%) y complementarlos en CSharp como comentario en el encabezado del programa**
   - Escribir el código en ensamblador ARM64 para cada uno de los 10 problemas.
   - El código debe estar **bien comentado** y seguir buenas prácticas de programación.

3. **Pruebas y Depuración (25%)**
   - Probar cada programa con diferentes casos de prueba para asegurar su correcto funcionamiento.
   - Documentar los **resultados de las pruebas** y cualquier proceso de depuración realizado.

4. **Demostración con ASCIINEMA (25%)**
   - Grabar una **animación con ASCIINEMA** mostrando la ejecución de cada programa.
   - La animación debe incluir la compilación, ejecución y resultados de cada programa.

---

**Instrucciones Detalladas:**

**Fase 1: Análisis y Diseño del Algoritmo**

- **Comprensión del Problema:**
  - Lee atentamente el enunciado de cada problema seleccionado.
  - Identifica los requerimientos y restricciones.

- **Diseño de la Solución:**
  - Elabora un **pseudocódigo** o **diagrama de flujo** que represente la lógica de tu solución.
  - Asegúrate de que tu algoritmo sea eficiente y efectivo.

**Fase 2: Implementación en Ensamblador ARM64**

- **Codificación:**
  - Transforma tu pseudocódigo o diagrama de flujo en código ensamblador ARM64.
  - Utiliza comentarios para explicar secciones clave de tu código.

- **Buenas Prácticas:**
  - Sigue convenciones de nomenclatura y estructura de código.
  - Optimiza tu código para mejorar el rendimiento cuando sea posible.

**Fase 3: Pruebas y Depuración**

- **Casos de Prueba:**
  - Diseña varios casos de prueba, incluyendo casos normales, extremos y erróneos.
  - Ejecuta tu programa con estos casos y verifica los resultados.

- **Depuración:**
  - Si encuentras errores, utiliza técnicas de depuración para solucionarlos.
  - Documenta cualquier cambio realizado durante este proceso.

**Fase 4: Demostración con ASCIINEMA**

- **Grabación:**
  - Utiliza ASCIINEMA para grabar la terminal mientras compilas y ejecutas cada programa.
  - Asegúrate de que el texto sea legible y que los comandos y resultados se muestren claramente.

- **Presentación:**
  - La animación debe mostrar:
    - **Compilación** del código.
    - **Ejecución** del programa.
    - **Salida** o resultados generados.

---

**Entrega Final:**

- Un archivo o carpeta que contenga:
  - **Pseudocódigos o diagramas de flujo** de cada problema.
  - Archivos de **código fuente** en ensamblador (.s).
  - Un documento con la **documentación de pruebas y depuración**, recordando que CShap esta comentado en el encabezado.
  - Enlaces o archivos de las **animaciones de ASCIINEMA**.

**Nota:** Asegúrate de que todos los archivos estén organizados y nombrados adecuadamente para facilitar la revisión.
